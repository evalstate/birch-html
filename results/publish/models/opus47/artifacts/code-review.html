Findings

High-risk findings first

H1 · high correctness test/eval

Eval runner compares the artifact against itself

evals/charts/run_eval.py · main() · lines 30–47 of new file

Evidence. Every --pair argument is built as "24-candle-chart-brief.html:24-candle-chart-brief.html". check_candle_renderings.compare_pair treats the left side as the source baseline and the right side as the candidate; passing the same path on both sides drives mean_abs_rgb_delta to 0, makes the amplified-diff PNG empty, and short-circuits every comparison-based check.

36+"--pair",
37+"24-candle-chart-brief.html:24-candle-chart-brief.html",

Impact. The eval is described as “exercises the Matplotlib SVG path and the Candle rendering checker at desktop and mobile viewports”. With identity pairs it only exercises the static-contract half of the checker; the screenshot-diff, palette-drift, and blackish-fraction warnings can never fire, so visual regressions land silently. Two of the four configured viewports are 390× mobile, which is precisely where the diff was most likely to catch new overflow.

Recommended fix. Check a known-good baseline render of 24-candle-chart-brief.html into evals/charts/baselines/ (or a fixture dir) and pair against that; alternatively, accept --baseline in run_eval.py and default to the latest committed PNG/HTML snapshot.

H2 · high silent failure parser

MetricRowParser only validates <div class="metric-row">

scripts/check_candle_renderings.py · MetricRowParser.handle_endtag

Evidence. The parser flips in_metric_row=True on any start tag with class metric-row, but the validation hook is gated on the closing tag being a </div>:

++if tag.lower() == "div" and self.depth == 0:
++ tags = [tag for tag, _classes in self.children]
++ ...validate three-child contract...

Impact. Any author writing <section class="metric-row">, <li class="metric-row">, or even a <span>-based row keeps in_metric_row latched after the first row ends, then validation never runs, and the parser’s children list accumulates across rows — producing both false-negatives and spurious failures on later rows. The sibling parsers (GridListParser, TimelineParser) correctly track the tag they opened on.

Recommended fix. Record the opening tag name when entering a metric-row and trigger validation when the closing tag matches and depth is zero. Reset children / preview in the same branch.

M1 · medium crash brief generator

previous_latest raises IndexError on a single-week dataset

evals/charts/build_chart_brief.py · previous_latest()

Evidence. weeks[-2], weeks[-1] is read without a length guard; previous_latest is then called unconditionally from main() and the result is interpolated into the KPI card (“Down from {prev_overflow} in the previous weekly sample.”).

++weeks = sorted({str(row["week"]) for row in rows})
++prev, latest = weeks[-2], weeks[-1]

Impact. Today’s sample-data.csv ships five weeks × three artifacts (16 rows incl. header), so the crash is latent. The first time the fixture is trimmed, reset for a fresh experiment, or filtered to a single week, the entire eval pipeline aborts before any HTML or rendering report is produced. The runner has check=True, so subsequent viewports also stop.

Recommended fix. Guard with if len(weeks) < 2: return ([], latest_rows) and render the “Down from N” copy conditionally, or compute the delta against a configurable baseline week.

M2 · medium repo hygiene

Chart brief writes to repo root with a hard-coded eval prefix

evals/charts/build_chart_brief.py · module-level OUT

++OUT = ROOT / "24-candle-chart-brief.html"

Impact. Every eval run drops a top-level 24-candle-chart-brief.html beside the project README, mixed in with source. The leading 24- looks like an external numbering scheme (eval index) that isn’t documented anywhere in the patch, and other generated outputs (reports/...) live under sensible subdirectories. The same hard-coded path is also embedded in run_eval.py’s --pair, so the two files must change together.

Recommended fix. Move output under reports/ or evals/charts/out/, drop the bare numeric prefix, and pass the path explicitly from run_eval.py so the two stay in sync.

L1 · low dead code

Pillow get_flattened_data() branch never runs

scripts/check_candle_renderings.py · palette_metrics() (context block in diff, lines ~767–790)

Evidence. resized.get_flattened_data() if hasattr(resized, "get_flattened_data") else resized.getdata(). PIL.Image.Image exposes getdata(), tobytes(), load(); there is no get_flattened_data on any Pillow image class.

Impact. Cosmetic — the else branch always wins. Worth pruning to avoid future readers chasing a phantom Pillow API.

Recommended fix. Replace with a direct call to resized.getdata() (or np.asarray(resized).reshape(-1, 3) if speed becomes a concern).

L2 · low portability

Generated HTML hard-links a relative stylesheet

evals/charts/build_chart_brief.py · template string in main()

++<link rel="stylesheet" href="styles/candle-system.css">

Impact. The artifact only renders correctly when opened from the repo root (or through the checker’s injected <base>). Anyone double-clicking the file from reports/ or sharing it standalone gets unstyled output. The checker was patched in the same diff to insert the <base> before the link — useful, but it does not help downstream consumers.

Recommended fix. Either inline the canonical Candle CSS into the artifact (mirroring the Birch placeholder pattern) or emit the file next to the stylesheet.

L3 · low licence

SPDX header is in the generated HTML but missing from the new Python sources

evals/charts/build_chart_brief.py, evals/charts/run_eval.py

Evidence. The HTML template starts with <!-- Copyright 2026 Anthropic PBC · SPDX-License-Identifier: Apache-2.0 -->; neither .py file carries the same header that other repository scripts conventionally include.

Recommended fix. Add an SPDX/copyright line to both new files to match repo norms.

Notable positives

What the patch gets right

  • The <base> placement bug is fixed correctly: a re.sub now inserts the base immediately after <head>, ahead of the relative stylesheet links. The in-source comment explains the prior symptom.
  • geometry_audit upgrades meaningfully: page-level horizontal overflow (pageOverflowX), timeline marker geometry, split-rail heading alignment, and numeric-table header alignment are all new and grounded in real Candle failure modes.
  • contract_findings escalates the right things to fail (grid-list children, metric-row contract, timeline-item contract, numeric-rich-but-chartless reports) instead of warn-only.
  • The chart brief uses exact CSV values for KPIs, table, and the ranking meter — satisfying the “evidence first” norm in the chart-brief recipe.

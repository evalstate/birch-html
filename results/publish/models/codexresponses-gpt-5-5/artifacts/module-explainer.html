Module explainer

Rendering checker pipeline

A source-grounded map for contributors changing scripts/check_birch_renderings.py, the chart fixture, or Birch HTML output rules without breaking existing rendering evals.

artifact mode pair mode desktop + mobile static + visual smoke
Mental model

The checker is a contract parser with optional browser evidence.

Think in three layers

  • Static contract layer. page_stats() parses HTML, counts tags/classes/data attributes, records CSS variables, and verifies that Birch CSS is embedded or linked.
  • Structural guard layer. Dedicated HTML parsers enforce fragile component contracts: .metric-row, .diff-row, .flow-step, timeline items, checklist rows, and grid list children.
  • Rendered evidence layer. When Chrome and Pillow are available, the checker captures screenshots, computes palette/blankness metrics, and runs an in-browser geometry audit for overflow and alignment defects.
Contributor rule of thumb

Add a deterministic parser or stat check for markup invariants. Add a geometry audit only when the failure is visual and cannot be proven from source text. Keep screenshots as evidence, not the only oracle.

Runtime path

From input pairs to reports, screenshots, and findings

Flow grounded in main(), check_artifact(), compare_pair(), contract_findings(), screenshot helpers, and geometry_audit() in scripts/check_birch_renderings.py. SVG uses Birch .flowchart, .flow-node, and .flow-edge classes and has title/description text for accessibility.

  1. 1

    CLI selects mode

    --artifact checks standalone generated candidates. --pair original:candidate compares a source/reference page to a Birch candidate. The script rejects mixed --artifact and --pair usage.

  2. 2

    Stats are extracted from HTML

    StatsParser feeds page_stats(), which records doctype, viewport, stylesheet links, embedded Birch CSS validity, style/script counts, tag counts, class counts, data attributes, text word counts, CSS variables, literal hex colors, and Birch CSS bytes.

  3. 3

    Contract findings are appended

    contract_findings() emits pass, warn, fail, or note entries for required shell, Birch CSS, page shell, primitive density, unknown CSS vars, gradients, SVG marks, and component child contracts.

  4. 4

    Rendered checks run when possible

    find_chrome() looks for Chrome or Chromium. With a browser, capture() writes PNG screenshots. With Pillow, artifact mode measures blankness and palette closeness; pair mode also computes mean/RMS RGB deltas and contact/diff sheets.

  5. 5

    Reports are written and failures exit nonzero

    main() writes JSON to --out and Markdown to --markdown, prints both paths, and exits with status 1 on any failure or on warnings when --fail-on-warn is set.

File tour

Read in this order before modifying behavior

  1. 1

    Start with the contract docs

    docs/birch-llm-style-guide.md defines the required shell, the exact Birch placeholder, allowed primitives, flow-step/diff/metric contracts, chart/SVG rules, and mobile safety constraints. styles/birch-system.css is the implementation of those primitives.

  2. 2

    Read the checker top down

    scripts/check_birch_renderings.py declares constants, dataclasses, parser classes, CLI entrypoint, mode routing, stats extraction, contract checks, capture helpers, geometry audit JavaScript, and Markdown rendering in one file.

  3. 3

    Use the chart eval as a working fixture

    evals/charts/run_eval.py regenerates 24-birch-chart-brief.html and validates it at desktop, mobile, and deep viewports. It is the clearest end-to-end example of expected commands and report paths.

  4. 4

    Inspect the artifact generator

    evals/charts/build_chart_brief.py loads CSV rows, converts numeric fields, builds a Matplotlib SVG via scripts/birch_mpl.py, emits Birch cards, metric rows, and a numeric table, then writes 24-birch-chart-brief.html.

  5. 5

    Only then touch chart styling helpers

    scripts/birch_mpl.py is deliberately small: rc_params(), birch_theme(), polish_axes(), svg_string(), and close(). Change this when chart SVGs should look different, not when checker behavior should change.

Commands to reproduce

uv run --with pillow python scripts/check_birch_renderings.py --artifact 24-birch-chart-brief.html --viewport mobile:390x900
uv run --with matplotlib --with pillow python evals/charts/run_eval.py

The chart runner first calls uv run --with matplotlib python evals/charts/build_chart_brief.py, then invokes the checker four times with named report paths.

Where behavior lives

Extension points and safe edit targets

New static rule

Add parser state or a simple assertion in contract_findings(). Use Finding(level, name, evidence) and keep evidence actionable.

  • Good fit: required child order, unknown CSS variables, forbidden gradients, missing Birch shell.
  • Also update docs/birch-llm-style-guide.md if the rule becomes contributor-facing.

New visual smoke rule

Add to geometry_audit() when browser layout is the only reliable proof. Convert audit payloads to findings in geometry_findings().

  • Good fit: overflow, metric meter alignment, table header alignment, split rail heading alignment.
  • Keep selectors bounded; the audit already scans common text, component, table, metric, and timeline targets.

New output/report field

Modify the payload assembled in main() and update render_markdown(). Artifact mode returns artifacts; pair mode returns pairs.

  • Maintain summary.failures, summary.warnings, and summary.notes because downstream summary scripts may depend on them.

Chart appearance

Change scripts/birch_mpl.py when Matplotlib SVGs should track Birch tokens. It controls palette, facecolors, fonts, axes, grids, legends, and SVG serialization cleanup.

  • Generated chart fixtures import birch_theme, polish_axes, svg_string, and close.
Gotchas

Failure modes that look harmless in code review

Artifact mode does not self-compare screenshots

The module docstring says generated eval artifacts normally use candidate-only --artifact mode. That mode captures rendered evidence and flags visible breakage, but true screenshot distance comparison only happens in --pair mode.

Deep viewports measure page height first

capture_height_for_viewport() only uses dynamic document height when the viewport name contains deep. Normal desktop/mobile captures use the supplied height.

Component parsers encode CSS assumptions

FlowStepParser and DiffRowParser exist because loose direct children become unintended grid columns. If CSS changes from grid to block layout, update both the style system and the checker rule together.

Standalone generated artifacts usually need embedded Birch CSS

contract_findings() accepts either valid embedded style[data-birch-system] or a link to styles/birch-system.css. The LLM contract prefers the placeholder plus postprocessing for standalone eval artifacts.

Checks worth preserving

  • doctype, viewport, and .page shell checks.
  • no_unknown_css_vars against system plus local definitions.
  • svg_chart_mark_presentation for chart paths and polylines.
  • metric_row_contract, diff_row_contract, and flow_step_contract.
  • blank_render, palette_drift, and overflow findings when capture works.
Report shape

Know what downstream readers see

The JSON payload is intentionally simple. In artifact mode, mode is artifact, artifacts contains one object per checked file, and summary.pairs is kept as a back-compat denominator even though it counts artifacts.

{
  "mode": "artifact",
  "artifacts": [
    {
      "artifact": "24-birch-chart-brief.html",
      "stats": { "title": "...", "class_counts": { "page": 1 } },
      "screenshot": { "png": "reports/birch-screenshots/24-birch-chart-brief-mobile.png" },
      "geometry": { "overflow": [] },
      "findings": [ { "level": "pass", "name": "doctype", "evidence": "doctype present" } ]
    }
  ],
  "summary": { "artifacts": 1, "pairs": 1, "failures": 0, "warnings": 0, "notes": 1 }
}

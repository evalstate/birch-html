B1 · layout_overflow upgraded from warn → fail

File: scripts/check_candle_renderings.py, function geometry_findings()

Severity: blocking Evidence: diff hunk

Evidence. The function previously returned Finding("warn", ...) for overflow. The patch promotes it to "fail" and adds a second fail condition for document-level horizontal overflow:

1157-"warn",
1158+"fail",
1169+findings.append(Finding("fail", "layout_overflow", f"document overflows viewport horizontally by {page_overflow_x}px"))

Impact. Any page with even 3px of text overflow (previously a warning) will now cause SystemExit(1) in main(). Existing CI pipelines that pass today will break on the next run against the same pages. The alert threshold (overX > 2) in the JS audit is tight—subpixel rounding alone can trip it.

Fix. Either keep the existing warn severity for the element-level overflow path and introduce the new document-level check as a separate warn, or gate this behind a new --fail-on-overflow flag (analogous to --fail-on-warn). Add a changelog note that existing users should expect new failures.

B3 · Division by zero in build_chart_brief.py

File: evals/charts/build_chart_brief.py, lines 151–154

Severity: blocking Evidence: source lines
151+avg_pass = sum(float(row["static_pass_rate"]) for row in latest) / len(latest)
153+avg_vision = sum(float(row["vision_score"]) for row in latest) / len(latest)
154+avg_delta = sum(float(row["mean_rgb_delta"]) for row in latest) / len(latest)

Impact. latest_rows() filters to the latest week. If sample-data.csv is empty or corrupted (zero rows for any week), len(latest) is 0 and main() raises ZeroDivisionError. The script is called unconditionally by run_eval.py before the rendering checker, so an empty CSV blocks the entire eval pipeline with an unhelpful traceback.

Fix. Guard with if not latest: sys.exit("No data for latest week") before the division. Optionally, add a similar guard in previous_latest() (line 105: weeks[-2] will IndexError if fewer than two weeks exist).

High

Behavioral shifts and fragile assumptions

H1 · geometry_audit() base-element insertion is a semantic fix with side effects

File: scripts/check_candle_renderings.py, function geometry_audit()

Severity: high Evidence: diff hunk

Evidence. The old code inserted <base> just before </head>, meaning it appeared after stylesheet <link> elements. Per the HTML spec, <base> only affects URLs in elements that follow it in tree order, so stylesheets linked earlier in <head> resolved against the temp-directory location—not ROOT—and silently failed to load. The new code inserts <base> immediately after <head>:

1109-if "</head>" in source:
1110- source = source.replace("</head>", base + audit_js + "</head>", 1)
1110+if re.search(r"<head(?:\s[^>]*)?>", source, flags=re.I):
1114+ source = re.sub(r"(<head(?:\s[^>]*)?>)", lambda m: m.group(1) + base, source, count=1, flags=re.I)

Impact. The fix is correct, but it changes the visual output of every prior geometry audit. Previous runs compared unstyled candidate pages; future runs will compare correctly styled pages. Layout overflow, timeline geometry, split-rail alignment, and metric-bar spread will all produce different results. Combined with B1 (overflow severity upgrade), the next CI run may produce a large number of new failures that are real (previously hidden by the base bug) but appear as a regression.

Fix. Accept the fix but pair it with a re-baseline of all existing artifacts. Run the checker with the new code against every page and triage failures before landing.

H2 · run_eval.py self-compare masks genuine drift

File: evals/charts/run_eval.py, line 42

Severity: high Evidence: source line
42+"24-candle-chart-brief.html:24-candle-chart-brief.html",

Impact. The --pair argument uses the same file for both original and candidate. In compare_stats(), this makes the “original” and “candidate” identical, so checks like text_coverage, section_count_drift, and component_density_drop always pass trivially. compare_pair() also skips screenshot capture when the paths are identical (since there is nothing to diff). The eval becomes a contract-only check, missing any visual regression that would be visible against a known-good baseline.

Fix. Either commit a golden 24-candle-chart-brief.baseline.html and pair against it, or add a dedicated --single mode that skips comparison checks and clearly reports “no baseline available” rather than silently passing them.

M2 · Audit JSON extraction regex may miss escaped quotes

File: scripts/check_candle_renderings.py, function geometry_audit(), around line 1130

Severity: medium Evidence: pattern analysis

Evidence. The audit JS writes a JSON blob into data-candle-audit, and the Python side extracts it with re.search(r'data-candle-audit="([^"]+)"', ...). If the JS audit captures text containing a literal double-quote character (e.g., from a textContent value like He said "hello"), the regex stops at that quote, producing truncated JSON that fails json.loads. The text() helper in the audit JS does not escape quotes.

Impact. Silent fallback: geometry_audit() returns None, and geometry_findings() logs a “browser geometry audit unavailable” note. Actual geometry problems go undetected.

Fix. Either escape double-quotes in the JS text() helper (.replace(/"/g, '&quot;')) or use a <script type="application/json" data-candle-audit> element and extract with re.search(r'<script[^>]*data-candle-audit[^>]*>(.*?)</script>', ...) plus html_lib.unescape.

M3 · SVG class injection regex is fragile

File: evals/charts/build_chart_brief.py, line 86

Severity: medium Evidence: source line
86+svg = re.sub(r"<svg\b", '<svg class="chart-svg" role="img" aria-label="Line chart showing weekly static pass rate by artifact"', svg, count=1)

Impact. If Matplotlib changes its SVG output (e.g., adds attributes before <svg or uses a different casing), the regex silently stops matching and the SVG loses its Birch class and accessibility label. The chart still renders visually, but it breaks the Birch component contract and the chart-svg class count in contract_findings().

Fix. Use an XML parser (e.g., xml.etree.ElementTree) to set attributes on the root element, or at minimum assert after the substitution that 'class="chart-svg"' in svg and fail loudly on mismatch.

Residual risks

After merge

  • Undiscovered broken pages. The four new fail checks in contract_findings() have not been run against the full artifact corpus. Until that is done, the blast radius is unknown. A page that passes today may fail tomorrow with no code changes of its own.
  • Geometry audit false positives. The viewport-overflow threshold in JS (offRight > 2 || offLeft > 2) may flag legitimate scrollbars, system font variations, or subpixel layout differences across Chrome versions.
  • Chart-brief data dependency. If evals/charts/sample-data.csv is regenerated with a different schema or fewer weeks, both build_chart_brief.py and the rendering checker will break in ways not covered by the existing test surface.
  • No screenshot diff on self-compare. Until a golden baseline is committed, the chart brief eval has no visual regression coverage—only contract and geometry checks.

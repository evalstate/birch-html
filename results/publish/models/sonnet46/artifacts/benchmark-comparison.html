Regressions & Improvements

Improvements

  • Score gains across all four evals (numeric-data +0.040, code-review +0.045, module-explainer +0.019, implementation-plan +0.088).
  • Checker warnings reduced 5→2 (−60 %), the dominant driver of the final score improvement.
  • code-review and implementation-plan evals now produce zero checker warnings.
  • No new checker failures introduced.

Regressions

  • Wall time increased 18.7 % (412 → 489 s). Difference includes model latency and harness overhead; treat as noisy at this scale.
  • Estimated cost increased 11.5 % ($1.92 → $2.14), driven by +7.5 % token growth (larger outputs). Accepted given quality gain.
  • Hygiene score dropped 1.000 → 0.990 (−0.010). Marginal; does not materially affect promotion decision.
  • module-explainer warning count unchanged (1 → 1). Mobile flow caption remains dense — carry forward as open action.
  • code-review artifact still needs manual inspection for long diff wrapping (flagged in decision_notes).
Caveats

Token Comparability & Checker Limitations

  • Token comparability: Provider token schemas differ. Cached, reasoning, and effective tokens are not apples-to-apples across runs or models. The +7.5 % token increase should be treated as directionally correct, not as a precise budget figure.
  • Checker warnings are mechanical: They validate contract signals (unsupported classes, external assets, CSS compliance) — not a complete human quality score. A lower warning count does not guarantee better prose, layout aesthetics, or factual accuracy.
  • Screenshot comparisons: Same:same screenshot pairs primarily validate geometry and rendering contracts, not aesthetic quality. Visual regressions that don't break contracts will not surface in checker warnings.
  • Wall time is noisy: Wall time includes model latency and harness overhead. Small differences (especially ±20 %) should be treated as noisy. Do not use wall time alone to make latency-critical decisions.
  • Single-model run: Results are from codexresponses.gpt-5.5 only. Cross-model validation (Kimi, Gemini) has not been performed; Candidate B's improvements may not generalize uniformly.
  • Eval coverage: Four artifact types (numeric-data, code-review, module-explainer, implementation-plan). Benchmark-comparison, design-directions, flow-diagram, and other types are not yet included in this eval set.

Checker upgrade: structural parsers, geometry expansion, severity escalation

This patch lands three new static HTML parsers, expands in-browser geometry auditing across four new check categories, and escalates layout_overflow from warn to fail. One blocking and one high-risk behavioral change require confirmation before merge; remaining issues are medium and low.

Severity · blocker

Breaking change: layout_overflow escalated to hard failure

Blocker
Blocker scripts/check_candle_renderings.py · geometry_findings()

layout_overflow severity: warn → fail

scripts/check_candle_renderings.py — geometry_findings(), overflow branch

Evidence
-Finding("warn", "layout_overflow", …)
+Finding("fail", "layout_overflow", …)
Impact

main() raises SystemExit(1) when payload["summary"]["failures"] > 0. Previously, overflow findings required --fail-on-warn to break the pipeline. After this change, any detectable overflow (including elements that are merely off-screen to the right but not scrollable) causes a hard failure. Existing canonical artifacts that previously passed may now fail without any HTML changes.

Additional context

The patch also adds offRight/offLeft detection in the overflow JS audit, broadening what triggers this now-fatal finding. An element only 3 px off the right edge of the viewport is sufficient.

Recommended fix
  • Confirm the escalation is intentional and communicate it to all users of the checker script.
  • Re-run run_eval.py and any existing CI jobs against all canonical artifacts before merge to identify breakage.
  • If the escalation is correct policy, add a --strict-overflow / --no-fail-overflow flag to allow opt-out for legacy contexts.
  • Update the checker's README or docstring to flag this as a breaking severity change.
Severity · high

Base-tag injection fix invalidates historical geometry baselines

High
High scripts/check_candle_renderings.py · geometry_audit()

Pre-patch geometry audits ran against unstyled HTML

scripts/check_candle_renderings.py — geometry_audit(), base-tag insertion block

Evidence — old insertion (base came after stylesheets)
-if "</head>" in source:
- source = source.replace("</head>", base + audit_js + "</head>", 1)
New insertion (base immediately after <head> open)
+if re.search(r"<head(?:\s[^>]*)?>", source, flags=re.I):
+ source = re.sub(r"(<head(?:\s[^>]*)?>)", lambda m: m.group(1) + base, …)
+ source = re.sub(r"</head>", lambda _m: audit_js + "</head>", …)
Impact

Relative stylesheet links (styles/candle-system.css) resolve against a file:///tmp/… path unless a <base> element precedes them. The old code inserted <base> immediately before </head>, i.e. after the stylesheet link; the browser had already resolved (and failed to load) CSS before the base href took effect. Headless Chrome was auditing plain HTML with browser-default styles. All historical overflow counts, metric-bar spreads, and timeline geometry results are unreliable and not comparable to post-patch runs.

The fix is correct

The new code is the right approach. However, consumers of stored audit JSON/Markdown reports should know pre-patch baselines are invalid. Recommend re-running all canonical artifact audits after merge and treating prior geometry pass/fail records as informational only.

Recommended fix
  • Accept the fix as correct — no code change needed.
  • Annotate or purge stored geometry audit reports generated before this commit.
  • Add a test that injects a known stylesheet link and verifies <base> appears before it in the modified source.

Medium scripts/check_candle_renderings.py · TimelineParser

assert isinstance(…) in HTML parser hot path — stripped with -O

scripts/check_candle_renderings.py — TimelineParser.handle_starttag() and handle_endtag()

Evidence
+children = item["children"]
+assert isinstance(children, list)
+children.append(classes)
+assert isinstance(children, list)
+assert isinstance(timeline_classes, set)
Impact

Python strips assert statements when run with -O. If the dict values are unexpectedly not a list/set, the subsequent .append() or set operation raises an unrelated AttributeError instead of a descriptive error. The values are always initialized as correct types in item_stack.append(), so this is low-likelihood but would produce a confusing traceback in a checker script.

Recommended fix
  • Replace assert isinstance(x, T) with if not isinstance(x, T): raise TypeError(…), or rely on type annotations and a type checker instead of runtime assertions in the hot path.
  • Alternatively, accept as-is if the project never runs checker scripts with -O.

Positive changes

Sound additions worth noting

Pass
  • Base tag fix is correct. Inserting <base> immediately after <head> open ensures shared CSS is loaded before Chrome runs any geometry audit. The old code was silently testing unstyled HTML.
  • New structural parsers are well-scoped. GridListParser, MetricRowParser, and TimelineParser validate specific Candle component contracts (grid-list child count, metric-row child structure, timeline-item required children) with distinct, enumerable failure modes.
  • Expanded overflow detection is more accurate. Adding offRight/offLeft via viewportWidth tracking catches elements that are clipped or off-screen without triggering scrollWidth overflow — a real class of 390 px mobile bugs the old audit missed.
  • Four new geometry checks fill real audit gaps. Timeline marker alignment, split-rail heading alignment, numeric table header/body alignment, and document-level pageOverflowX all have practical impact on Candle artifact quality.
  • build_chart_brief.py follows Candle contracts correctly. The metric_rows() function produces strict .caption + .meter + code children, the HTML output uses .page.stack as the outer shell, and all color tokens reference var(--success) / var(--accent) (not hardcoded hex values).

Residual risks

Open questions after merge

  • Geometry audit flakiness. The headless Chrome path (--virtual-time-budget=800) may produce inconsistent results for SVG-heavy pages or slow CSS animations. The new timeline-marker and split-rail checks depend on precise pixel offsets; flakiness tolerance thresholds (2.5 px, 3 px, 8 px) are not tested across viewport densities (HiDPI).
  • Cumulative false-positive rate. Adding four new geometry checks and three new structural-parser checks increases the total number of fail-capable checks. The aggregate false-positive rate of the checker against legitimately correct Candle artifacts is not characterised in this diff.
  • MetricRowParser depth tracking on non-div metric-row containers. The validation logic in handle_endtag fires when tag == "div" and depth == 0. If a .metric-row were implemented on a non-div element (e.g., <li class="metric-row">), the metric-row would never close in the parser's view and in_metric_row would remain True until the next metric-row opens. Currently this appears consistent with Candle usage, but is worth noting.
  • No integration test between the three new files. build_chart_brief.py generates HTML that will be checked by check_candle_renderings.py via run_eval.py. A failure in any new check targeting the chart-brief output would be caught at eval time, but is not verified in this diff.

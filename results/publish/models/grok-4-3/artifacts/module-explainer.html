Rendering Checker Pipeline

Mental model, runtime path, file tour, and extension points for scripts/check_birch_renderings.py

Mental Model

The checker is a static-first contract validator that also supports optional visual regression. It parses HTML artifacts, enforces Birch class contracts and layout invariants, then optionally captures screenshots and computes perceptual diffs. It is deliberately deterministic and project-local (no external services).

Core invariant: every Birch HTML artifact must declare the system stylesheet via <style data-birch-system> and use only documented layout/semantic classes. Violations become Findings with level, name, and evidence.

Runtime Path: Input Pairs → Reports / Screenshots / Findings

  1. Entry — main() parses argparse for --pair, --artifact, --viewport, --chrome, --metrics.
  2. Static pass — StatsParser + specialized parsers (GridListParser, ChecklistParser, MetricRowParser, DiffRowParser) walk every tag/class.
  3. Contract checks — Enforce BIRCH_CLASSES, SEMANTIC_CLASS_MIN, BIRCH_CSS_MIN_BYTES, palette, and component child contracts.
  4. Optional capture — If --chrome, spawn headless Chrome at each viewport listed in VIEWPORTS (see evals/charts/run_eval.py).
  5. Optional metrics — Pillow compares screenshots; records Euclidean distance and palette delta into Finding objects.
  6. Emit — Write eval-runs/.../report.json containing PageStats + list of Finding and optional screenshot paths.

File Tour (Recommended Reading Order)

  1. scripts/check_birch_renderings.py — Entire pipeline; start here. Read StatsParser, Finding, PageStats, then each contract parser.
  2. scripts/birch_mpl.py — Matplotlib theme helpers used by chart fixtures. Shows how SVG charts are produced for later checking.
  3. evals/charts/run_eval.py — Shows the public API: VIEWPORTS dict, run() helper, and how the checker is invoked from an eval.
  4. evals/charts/build_chart_brief.py — Concrete example of an artifact that the checker validates (uses birch_mpl).
  5. docs/birch-llm-style-guide.md — Authoritative list of allowed classes and the data-birch-system contract.
  6. styles/birch-system.css — Ground truth for tokens and component styles referenced by the checker.

Gotchas

  • Anonymous grid children — Loose text or extra <li> chunks inside .insight-list / .takeaway-list become unexpected grid items. GridListParser catches this.
  • Metric-row contract — Exactly three direct children: .caption, .meter, <code>. Any deviation is a hard failure.
  • Diff-row contract — Must be .ln + .mark + .code. Text outside these three columns collapses the layout.
  • --artifact mode does not compare screenshots; it only captures evidence. True regression requires --pair with a reference.
  • Chrome screenshots are written next to the HTML artifact with viewport suffixes (e.g. -desktop.png).

Concrete Commands & Outputs

# Run checker on a single artifact (candidate-only mode)
python scripts/check_birch_renderings.py --artifact 24-birch-chart-brief.html --chrome --metrics

# Run full pair comparison (reference vs candidate)
python scripts/check_birch_renderings.py --pair 05-design-system.html 05-design-system-birch.html

# Typical eval invocation (from evals/charts)
python evals/charts/run_eval.py

# Output locations
eval-runs/<run-id>/report.json          # Findings + PageStats
eval-runs/<run-id>/*.png                # Screenshots (desktop / mobile)
eval-runs/<run-id>/<artifact>-diff.json # Optional perceptual metrics

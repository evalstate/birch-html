# Module explainer rubric

Score each category 1–5.

1. **Code-grounded mental model** — accurately explains the checker's purpose and major moving parts.
2. **Runtime path** — traces inputs, parsing/checking, screenshot/render steps, metrics, and output reports.
3. **File tour** — gives concrete file paths in a useful reading/modification order.
4. **Flow quality** — diagram/list is accurate, not decorative, and mobile-safe.
5. **Contributor utility** — includes gotchas, extension points, commands, and where to change behavior.
6. **Birch contract** — uses `.split`, `.section-rail`, `.reference-panel`, `.flow-list`, `.flow-step`, `.code-block`, `.card`, etc.; avoids invalid classes.

Fail conditions: no source-file references, invented APIs, unreadable diagram, external assets, unknown classes/variables.

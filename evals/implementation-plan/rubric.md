# Implementation plan rubric

Score each category 1–5.

1. **Sequencing judgment** — gives a clear first step and explains why.
2. **Source fidelity** — accurately reflects `migration-plan.md`, including outstanding questions and phase intent.
3. **Known vs assumed** — separates confirmed facts, assumptions, open questions, and non-goals.
4. **Milestones and exits** — phases have concrete changes and validation/exit criteria.
5. **Verification plan** — includes 3–4 before/after evals and how to compare Birchline, Birch-before, Birch-after.
6. **Birch contract/readability** — uses appropriate Birch primitives (`.flow-list`, `.callout`, `.checklist`, `.plain-list`, `.numeric-table`/table, `.card`) and remains scannable/mobile-safe.

Fail conditions: no explicit first step, generic project-management filler, missing eval plan, unknown/Birchline-only classes.
